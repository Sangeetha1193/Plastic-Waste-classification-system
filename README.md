# WasteVision — Plastic Waste Classifier (YOLOv8 + Flask + Docker + AWS)

WasteVision detects common waste items from an image (upload or webcam frame) using a **YOLOv8 object detection model (17 classes)** and returns **bounding boxes + recycling tips** via a Flask API and web UI.

## Highlights
- **Web UI**: Webcam + upload modes with live overlay boxes + detection cards
- **Flask API**: `POST /predict`, `GET /health`, `GET /metrics`
- **Dockerized**: one-command local run with `docker compose`
- **Evaluation**: scripts to compute mAP/precision/recall and save PR + confusion matrix plots
- **AWS-ready**: manual EC2 deployment steps (first deployment, pre-CI/CD)

## Demo (local)
- UI: `http://localhost:5000/`
- Health: `http://localhost:5000/health`

## Model + classes
- **Weights**: `models/best.pt`
- **Dataset format**: YOLO
- **Taxonomy**: reduced to **17 classes** (used during training + evaluation)

## API

### `GET /health`
Returns server + model readiness.

Example:
```bash
curl http://localhost:5000/health
```

### `POST /predict`
Upload an image and get detections.

- **Body**: `multipart/form-data` with `image=<file>`
- **Query params**:
  - `conf` (float, default `0.25`)
  - `iou`  (float, default `0.45`)

Example:
```bash
curl -F "image=@test.jpg" "http://localhost:5000/predict?conf=0.25&iou=0.45"
```

Response includes:
- `predictions[].class_name`
- `predictions[].confidence`
- `predictions[].bbox` (pixel + normalized coords)
- `predictions[].recycling` (tip/bin/colour)

### `GET /metrics`
Returns model metadata (optionally reads `outputs/metrics.json` if present).

## AWS deployment (manual, first deployment using Docker on EC2)
This is a simple, “no CI/CD yet” deployment using **EC2 + Docker Compose**.

### 1) Launch EC2
- AMI: **Ubuntu 22.04**
- Instance: choose enough CPU/RAM for YOLO inference (CPU-only inference benefits from more headroom)
- Storage: **30 GB** recommended (images + dependencies can be large)
- Security Group inbound:
  - `22` (SSH) from **your IP**
  - `5000` (HTTP app) from your IP (quick testing)
  - (recommended later) `80`/`443` with Nginx + TLS, and keep `5000` private

### 2) SSH into EC2 (Windows example)
```bash
ssh -i "C:\\Users\\<you>\\.ssh\\plastic_waste.pem" ubuntu@<EC2_PUBLIC_IP>
```

### 3) Install Docker + Compose on EC2
```bash
sudo apt-get update -y
sudo apt-get install -y ca-certificates curl gnupg

sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
$(. /etc/os-release && echo \"$VERSION_CODENAME\") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt-get update -y
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo systemctl enable --now docker

sudo usermod -aG docker ubuntu
newgrp docker
docker --version
docker compose version
```

### 4) Upload the project to EC2
From your local machine:
```bash
scp -i "C:\\Users\\<you>\\.ssh\\plastic_waste.pem" -r "Plastic-Waste-classification-system" ubuntu@<EC2_PUBLIC_IP>:/home/ubuntu/
```

### 5) Build + run on EC2
On EC2:
```bash
cd /home/ubuntu/Plastic-Waste-classification-system
docker compose up --build -d
docker compose ps
curl http://localhost:5000/health
```

### 6) Access from your laptop
If Security Group allows inbound `5000`:
- UI: `http://<EC2_PUBLIC_IP>:5000/`
- Health: `http://<EC2_PUBLIC_IP>:5000/health`

## Project structure (key files)
- `app.py` — Flask API + YOLO inference
- `templates/index.html` — web UI
- `models/best.pt` — trained weights
- `evaluate.py` — evaluation on val/test (plots + metrics)
- `visualize.py` — save or show annotated predictions
- `docker-compose.yml` — local run
- `Dockerfile` — container build

## Troubleshooting
- **Port already in use**: change compose mapping to `5001:5000`, or stop the process using 5000.
- **Docker build is huge**: ensure `requirements.txt` uses CPU wheels (`torch==...+cpu`).
- **Webcam blocked**: use `http://localhost` or deploy behind HTTPS.
